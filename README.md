# La Tua Pasta — London Restaurant Prospecting Tool

Internal web app for finding, scoring, and contacting London restaurant prospects
for La Tua Pasta (fresh-pasta trade supply). Built per `PRD.md`.

## Stack
- **Next.js 14** (App Router) + **TypeScript** + **Tailwind**
- **Leaflet + OpenStreetMap** map (clustered, ~20k venues)
- **Food Standards Agency** data (`public/london-restaurants.json`, generated by `scripts/fetch-fsa.mjs`)
- **Anthropic Claude** assistant (filter, add customers, draft emails, scan openings)
- **Supabase Postgres** for shared team data (optional — falls back to localStorage)
- **Shared-password** access gate

## Run locally
```bash
npm install
cp .env.example .env.local   # then fill in values (see below)
npm run dev                   # http://localhost:3000
```
Log in with the value of `SITE_PASSWORD` (default `latuapasta` if unset).

## Environment variables (`.env.local` and Vercel)
| Var | What | Required |
|---|---|---|
| `SITE_PASSWORD` | The shared access password you hand out | Yes (in prod) |
| `ANTHROPIC_API_KEY` | Powers the assistant + email drafting + web scan | For AI features |
| `ANTHROPIC_MODEL` | Optional model override (default `claude-opus-4-8`; `claude-haiku-4-5` is faster/cheaper) | No |
| `SUPABASE_URL` | Supabase project URL | For shared data |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase **service role** key (server-only secret) | For shared data |

Without the Supabase vars the app still works, but each browser keeps its own
added customers/edits (the sidebar shows "Local only"). With them it shows
"Shared team data".

## Make it public — deploy to Vercel
1. **Supabase (shared data):**
   - Create a free project at supabase.com.
   - SQL Editor → paste `supabase-schema.sql` → Run.
   - Project Settings → API → copy the **Project URL** and the **service_role** key.
2. **Push to GitHub** (this folder).
3. **Vercel:** New Project → import the repo → Framework: Next.js → Deploy.
4. **Vercel → Settings → Environment Variables:** add `SITE_PASSWORD`,
   `ANTHROPIC_API_KEY`, `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` (and
   optionally `ANTHROPIC_MODEL`). Redeploy.
5. Share the Vercel URL + the password with whoever you want to have access.

## Auth note
Access is a single shared password (`SITE_PASSWORD`), gated server-side
(`/api/login`) + route middleware. To move to individual accounts +
registration later, swap the password gate for Supabase Auth — the app structure
is ready for it.

## Refresh the venue dataset
```bash
node scripts/fetch-fsa.mjs   # re-pulls FSA data into public/london-restaurants.json
```
